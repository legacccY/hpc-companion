# HPC Companion 使用教程

把「连服务器、提交任务、看显卡、传文件」这些命令行操作，变成点几下鼠标就能完成的桌面程序。专为不熟悉 Linux / SLURM 的同学准备。

支持任意标准 SLURM 集群，自带常用 GPU 集群字段的一键预设。

## 一、它能帮你做什么

用集群本来要记一堆命令，新手很容易卡住。这个软件把它们都做成了界面：

| 你想做的事 | 命令行做法 | 用本软件 |
| --- | --- | --- |
| 连上集群 | `ssh user@host`，手动输密码 | 选预设、填一次密码，自动记住 |
| 看任务状态 | 反复敲 `squeue` | 表格自动刷新，状态有颜色 |
| 看显卡用满没 | 登进节点敲 `nvidia-smi` | 实时曲线，像 Windows 任务管理器 |
| 传文件 | 记 `scp` 路径 | 左右双栏，选中即传 |
| 提交任务 | 手写 `submit.sh` + `sbatch` | 填表单，自动生成脚本 |

全程不用记任何命令。

## 二、安装与启动

**方式 A — 直接下载现成版本（推荐，给非开发者）**

到项目的 [Releases 页](../../releases) 下载：

- **Windows** —— `HPC-Companion-windows.exe`，双击打开，无需装 Python。
- **macOS** —— `HPC-Companion-macos.zip`，解压得到 `HPC Companion.app`，拖进「应用程序」。首次打开若提示「无法验证开发者」，**右键 App → 打开**，或到「系统设置 → 隐私与安全性」点「仍要打开」（App 没做 Apple 付费签名，属正常）。

**方式 B — 源码运行（开发 / 自己改，Windows / macOS / Linux 通用）**

```bash
pip install -r requirements.txt
python main.py
```

> ⚠️ PyQt6 版本锁定 `6.6.1`。`6.7+` 在部分 Anaconda / 旧 MSVC 运行库环境会报
> `DLL load failed while importing QtCore`，换环境前先确认。macOS / Linux 一般无此问题。

## 三、第一次使用：配置连接

第一次打开会停在「连接」页。

![连接页](images/01_连接.png)

*连接页：左边是已保存的集群列表，右边填连接信息。*

操作步骤：

1. **先连 VPN**。多数学校集群只能在校园网 / VPN 内访问，校外不连 VPN 会提示「无法连接主机」。
2. 在「**套用预设**」里选示例预设，分区、QOS 等常用字段会自动填好；主机、用户名、account 填你自己的，或点「新建」从零配置。
3. 填**用户名**和**密码**。
4. （建议）勾选「**下次启动自动连接此集群**」，以后打开软件会自动连、直接进任务监控。
5. 点「**保存并连接**」。顶部状态条变绿、显示 `已连接：用户名@主机`，就成功了。

**关于密码安全**

- 密码存进**系统凭据库**（Windows 凭据管理器 / macOS Keychain），配置文件里只存非敏感信息，**绝不明文落盘**。
- 凭据库不可用时退化为「本次运行内存保存」，重启需重填——同样不写明文到硬盘。

> 配置过一次后，之后每次打开软件（若勾了自动连接）会自动连上并跳到「任务监控」，不用再操作连接页。

## 四、看任务跑得怎么样（任务监控）

连上后自动进入「任务监控」页，这是用得最多的一页。

![任务监控](images/02_任务监控.png)

*任务监控：表格自动刷新，状态用颜色区分；下方是选中任务的详情。*

**任务表格**

- **自动刷新**：默认 10 秒一次，右上「间隔」可调，最快 **1 秒**。
- **状态有颜色**：运行中=绿、排队=黄、失败=红、已完成=灰，一眼看清。
- **结束的任务不消失**：勾「显示已结束」并选时间范围（今日 / 近 3·7·30 天），跑完的任务照样能回看（`squeue` 活跃任务 + `sacct` 历史自动融合）。
- **一键取消**：选中某行 → 点右上「取消选中任务」→ 确认（相当于 `scancel`）。

**下方详情区**（跟着选中的任务实时更新）

- **日志**：自动定位日志文件并显示末尾内容，可切「标准输出 (.out)」/「错误输出 (.err)」。
- **GPU / 显存**：实时显卡曲线，见[下一节](#五实时显卡曲线gpu--显存)。
- **scontrol 详情**：任务的完整属性（节点、时限、工作目录等）。

> 任务失败时，详情区顶部会弹**红色横幅**，写明退出码（如 `127` 命令未找到、`137 / 139` 内存超限被杀）和错误日志末尾几行，不用自己翻日志找原因。

## 五、实时显卡曲线（GPU / 显存）

选中一个**正在运行**的 GPU 任务，点详情区的「GPU / 显存」子页：

![GPU 显存两栏曲线](images/03_gpu显存.png)

*左栏 GPU 利用率、右栏显存利用率，像任务管理器一样从右往左滚动。*

- **左栏 GPU 利用率（SM %）**：计算核心忙不忙，跑满训练时应接近 100%。
- **右栏 显存利用率（Mem %）**：显存读写有多忙。
- 顶部一行给出峰值、均值、显存占用 MB。

曲线做成了 **Windows 任务管理器那种观感**：纵轴锁定 0–100% 不乱缩放；固定时间窗**从右往左滚动**（新数据从右进、旧的左移出窗）；曲线下半透明填充 + 平滑处理，不是生硬的折线。

> 💡 **想看得更密**：把刷新间隔调到「1 秒」。曲线缓冲有上限，1 秒刷新也不会让内存越堆越多。
>
> 💡 **省资源**：显卡数据只在你正看着这一页时才采集，切到别的页就停；上一次还没采完也不会重复发——快刷新时不会卡。

结束的任务没有实时数据，会提示「仅运行中的 GPU 任务有利用率数据」。

## 六、传文件（本地 ↔ 集群）

切到「文件传输」页，左右双栏：左边本地、右边集群。

![文件传输](images/04_文件传输.png)

*双栏浏览，多选文件上传 / 下载，带进度条。*

- 两栏各自浏览目录，多选文件后**上传 / 下载带进度条**（显示速度、剩余时间）。
- 大文件**断点续传**：传一半断了，重传从断点接着传，不从头来。
- 集群侧可新建目录 / 删除文件。

## 七、提交任务（填表就能跑）

不用手写脚本。切到「提交任务」页：

![提交任务](images/05_提交任务.png)

*左边填表，右边实时生成脚本，可手动编辑，一键提交。*

1. **左边填表**：作业名、节点 / CPU / GPU 数、时限、工作目录、执行命令等。连接后 account、分区、QOS、工作目录会**自动预填**。
2. **右边实时预览** `submit.sh`：表单一改就重新生成。预览框**可以直接手动编辑**——手改后提交就用你改的原文（所见即所得）；想恢复点「↻ 从表单重新生成」。
3. 点「**写入远端并 sbatch 提交**」→ 确认，软件把脚本写到集群并提交，弹窗返回任务编号（Job ID）。
4. 回「任务监控」就能看到它排队 → 运行。

> 已经有现成脚本？直接把整段粘进右边预览框（会切到「手动编辑」模式），填好工作目录和脚本路径再提交即可。

## 八、常见问题

| 现象 | 原因 / 解决 |
| --- | --- |
| 连接报「无法连接主机」 | 校外没连 VPN，或主机地址 / 端口不对。先连 VPN 再试。 |
| 连接报「认证失败」 | 账号或密码错误；也可能是没连 VPN 导致到不了主机。 |
| 启动报 `DLL load failed while importing QtCore` | PyQt6 版本问题，锁回 `6.6.1`（见[第二节](#二安装与启动)）。 |
| GPU 曲线一直空白 | 选的任务不是运行中、或不是 GPU 任务；结束的任务没有实时数据。 |
| 长时间挂着会不会掉线 | 连接每 30 秒发一次保活心跳，防止被防火墙掐断。 |
| 界面会不会卡 | 所有网络操作都在后台线程跑，界面始终不冻结。 |

## 九、自己打包（Windows / macOS / Linux）

同一份 `build.spec` 跨平台，在对应系统上运行即可：

```bash
pip install pyinstaller
pyinstaller build.spec --noconfirm
```

| 系统 | 产物 |
| --- | --- |
| Windows | `dist/HPC-Companion.exe`（单文件） |
| macOS | `dist/HPC Companion.app`（拖进「应用程序」） |
| Linux | `dist/HPC-Companion`（单文件可执行） |

`build.spec` 按平台收 keyring 后端、处理 paramiko / cryptography 隐式导入与 OpenSSL 兼容；macOS 额外包成 `.app`。

> 💡 **没有 Mac 也能出 macOS 版**：推一个 `v*` tag，仓库的 GitHub Actions 会在云端 Windows / macOS runner 上自动打包并发布到 Releases。

首次打包后，建议在一台没装 Python 的干净机器上验证：能启动、能连接、密码能存进系统凭据库。
